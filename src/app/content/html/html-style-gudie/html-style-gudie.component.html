HTML Style Guide

Consistent, clean, and tidy HTML code makes it easier for others to read and understand your code.

Here are some guidelines and tips for creating good HTML code.

Always Declare Document Type

A semantic element clearly describes its meaning to both the browser and the developer.

The correct document type for HTML is:

Example

<!DOCTYPE html>

Use Lowercase Element Names

HTML allows mixing uppercase and lowercase letters in element name

However, we recommend using lowercase element names, because:

  • Mixing uppercase and lowercase names looks bad
  • Developers normally use lowercase names
  • Lowercase looks cleaner
  • Lowercase is easier to type

Good:

<body >
<p > This is a paragraph. </p >
</body >

HTML allows attribute values without quotes.

  • Quoted values are easier to read

Always Specify alt, width, and height for Images

Always specify the alt attribute for images. This attribute is important if the image for some reason cannot be displayed.

Also, always define the width and height of images. This reduces flickering, because the browser can reserve space for the image before loading.

Good:

<img src ="html5.gif" alt ="HTML5" style ="width:128px;height:128px" >

Avoid Long Code Lines

When using an HTML editor, it is NOT convenient to scroll right and left to read the HTML code.

Avoid too long code lines.

Never Skip the <title> Element

The <title> element is required in HTML.

The page title plays a crucial role in search engine optimization (SEO). Search engines use it to help determine the ranking and relevance of a page in search results.

The <title> element:

  • defines a title in the browser toolbar
  • provides a title for the page when it is added to favorites
  • displays a title for the page in search-engine results
<title > HTML Style Guide and Coding Conventions </title >

Omitting <html> and <body>?

An HTML page will validate without the <html> and <body> tags:

Example

<!DOCTYPE html>

  <title > Page Title </title >
</head >

<h1 > This is a heading </h1 >
<p > This is a paragraph. </p >

However, we strongly recommend to always add the <html> and <body> tags!

Omitting <body> can produce errors in older browsers.

You should always include the lang attribute inside the <html> tag, to declare the language of the Web page. This is meant to assist search engines and browsers.

Meta Data

To ensure proper interpretation and correct search engine indexing, both the language and the character encoding <meta charset="charset"> should be defined as early as possible in an HTML document:

Example

<html lang ="en-us" >

  <meta charset ="UTF-8" >

</head >

Setting The Viewport

The viewport is the user's visible area of a web page. It varies with the device - it will be smaller on a mobile phone than on a computer screen.

You should include the following <meta> element in all your web pages:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This gives the browser instructions on how to control the page's dimensions and scaling.

The width=device-width part sets the width of the page to follow the screen-width of the device (which will vary depending on the device).

HTML Comments

Short comments should be written on one line, like this:

<!-- This is a comment -->

Comments that spans more than one line, should be written like this:

<!--
  This is a long comment example. This is a long comment example.
  This is a long comment example. This is a long comment example.
-->

Using Style Sheets

<link rel ="stylesheet" href ="styles.css" >

Short CSS rules can be written compressed, like this:

p.intro { font-family :Verdana ; font-size :16em ; }

Long CSS rules should be written over multiple lines:

body {
  background-color : lightgrey ;
  font-family : "Arial Black", Helvetica, sans-serif ;
  font-size : 16em ;
  color : black ;
}
  • Place the opening bracket on the same line as the selector
  • Use one space before the opening bracket
  • Use two spaces of indentation
  • Use semicolon after each property-value pair, including the last
  • Only use quotes around values if the value contains spaces
  • Place the closing bracket on a new line, without leading spaces

Loading JavaScript in HTML

Use simple syntax for loading external scripts (the type attribute is not necessary):

<script src ="myscript.js" >

Accessing HTML Elements with JavaScript

Using "untidy" HTML code can result in JavaScript errors.

These two JavaScript statements will produce different results:

getElementById( "Demo"). innerHTML = "Hello";

getElementById( "demo"). innerHTML = "Hello";

Use Lower Case File Name

Certain web servers, like those running on Apache or Unix systems, are case sensitive with file names. For example, "image.png" and "Image.png" would be treated as two different files.

Some web servers, such as those using Microsoft IIS, are not case sensitive. This means a file named "london.jpg" can still be accessed using "London.jpg" or other variations in letter casing.

If you use both uppercase and lowercase letters in file names, you need to be mindful of how they’re referenced, especially on case-sensitive servers.

Switching from a case-insensitive to a case-sensitive server can cause your website to break, as even minor differences in letter casing can lead to missing files or broken links.

File Extensions

HTML files should use the .html extension, although .htm is also acceptable and still recognized by most browsers.

CSS files must use the .css extension to be properly recognized and applied by web browsers.

JavaScript files should use the .js extension so that browsers can correctly identify and execute the script.

Differences Between .htm and .html?

The .htm and .html file extensions are functionally the same—there’s no difference between them in how browsers handle the files.

Web browsers and servers recognize both extensions as HTML files, so they will handle .htm and .html in the same way.

Default Filenames

If a URL doesn’t include a specific file name (like "https://www.example.com/"), the server automatically looks for a default file, such as "index.html", "index.htm", "default.html", or "default.htm".

If your server is set up to use only "index.html" as the default file, then your homepage must be named "index.html"—naming it "default.html" won't work.
